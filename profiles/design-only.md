# Design-only workshop

Use this when you are running a **design-only** session (no frontend code). You get core principles and design doc templates only; no frontend rules.

## Rules to copy

From this repo into your project’s `.cursor/rules/`:

- `core/design-core.mdc`
- `core/cursor-behavior-constraints.mdc`

That’s it. No `frontend/*` or `binders/*` unless you add them later.

## Design docs to copy

Copy into your project’s `design/` folder and rename (drop `.example`):

- `design/tokens/colors.example.md` → `design/tokens/colors.md`
- `design/tokens/typography.example.md` → `design/tokens/typography.md`
- `design/tokens/spacing.example.md` → `design/tokens/spacing.md`
- `design/ia/navigation.example.md` → `design/ia/navigation.md`
- `design/content/voice-and-tone.example.md` → `design/content/voice-and-tone.md`

Optional: `design/research/findings.example.md` → `design/research/findings.md` (or similar).
